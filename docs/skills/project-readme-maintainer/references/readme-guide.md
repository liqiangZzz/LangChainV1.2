# README 维护细则

> 这是 `project-readme-maintainer` skill 的参考文档，服务对象是项目根目录 `README.md`。

## README 的职责

README 是给项目使用者和学习者看的入口说明，应回答：

- 这个项目是什么。
- 需要什么环境变量。
- 从哪里开始运行。
- 主要目录分别学什么。
- 哪些脚本会触发真实 LLM 调用和 API 额度消耗。

README 不应该承担：

- 每个示例文件的完整教程。
- 未实现技术点的完整规划。
- 包级 `__init__.py` 的详细目录说明。
- 密钥、Token、密码等敏感信息展示。

## 推荐结构

```markdown
# LangChainV1.2

## 项目简介

## 目录结构

## 环境配置

## 快速开始

## 学习模块

## 运行注意事项

## 文档维护
```

## 写作规则

- 默认使用中文。
- 只写已有代码承载的内容。
- 优先增量维护，保留 README 既有结构、语气和已有有效内容。
- 新增模块时，只补充相关目录、学习模块或运行注意事项，不重写整份 README。
- 删除或大改已有 README 内容前，必须确认它已经和当前项目事实冲突。
- 如果某个模块还没有示例代码，不写成已完成内容。
- 目录说明优先引用已有包说明，例如 `models/__init__.py` 和 `agent_part/__init__.py`。
- 环境变量只写变量名，不写真实值。
- 涉及真实模型调用时，提醒会消耗 API 额度。
- 涉及 `deepseek-reasoner` 时，提醒推理模型通常成本更高。

## 增量维护判断

- README 不存在：可以创建一份基础 README。
- README 已存在且结构合理：只更新受影响的小节。
- 新增 Python 包：优先根据该包 `__init__.py` 补充目录结构或学习模块。
- 新增子包：先维护子包和上级包的 `__init__.py`，再判断它是否重要到需要进入 README。
- 新增入口脚本：补充快速开始或运行示例。
- 新增真实 LLM 调用或推理模型示例：补充运行注意事项。
- 仅文案风格不完美：不为了美化而大范围重写。

## AGENTS.md 的边界

- `AGENTS.md` 描述自动化编程助手在整个仓库中的协作规则，不是模块索引。
- 普通新增模块、示例文件或运行命令，不需要同步更新 `AGENTS.md`。
- 只有项目技术栈、目录约定、文档维护流程、安全规则或验证方式发生变化时，
  才增量更新 `AGENTS.md`。

## 和 package-init-doc-maintainer 的区别

- `project-readme-maintainer`
  维护根目录 `README.md`，面向项目入口说明。

- `package-init-doc-maintainer`
  维护各 Python 包的 `__init__.py`，面向包级目录说明。
